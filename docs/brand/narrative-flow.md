## CT-2: 訪問者の心理変化（時間軸）

### 5秒: IMPRESSION（印象）

**何が起きるか:** ビジュアルとHeroコピーが視界に入る

**訪問者の心理状態:**
- 「何の会社？」という疑問が自動的に生まれる
- ビジュアルのトーンで「感情的な文脈」が設定される
- 最初の数語でタグ付けされる（tech? design? Japanese? AI?）

**達成すべき感情状態:** 「なんか他のテック企業と違う」

**リスク:** 「またAI系スタートアップか」と判断されて離脱
**対策:** "縁"という日本語の言葉が持つ固有の温度感で差別化する

**設計判断:**
- Hero H1は「縁を、深く。」のみ。説明しすぎない
- ビジュアルは冷たくない（暖色・有機的・人間を感じる）
- 英語ではなく日本語を主言語として前面に出す

---

### 30秒: ORIENTATION（方向把握）

**何が起きるか:** サブヘッド + Featured Projects が視野に入る

**訪問者の心理状態:**
- 「ああ、アプリ/プロダクトを作っている会社か」
- 「Family Quest? ご縁ノート? 面白そう」
- 「テクノロジーと人間の繋がり、というテーマがあるんだ」

**達成すべき理解:** 「ENISHIはプロダクト会社で、人と人の繋がりをテーマにしている」

**リスク:** プロダクトが何かわからず離脱
**対策:** ProjectCard に tagline を大きく。一目でわかる thumbnail

---

### 2分: BELIEF（信頼形成）

**何が起きるか:** Visionセクション + About Snippet を読む。または /about へ遷移

**訪問者の心理状態:**
- 「この会社、ちゃんと考えてる」
- 「創業者のストーリーが共感できる」
- 「プロダクトの背後に哲学がある」

**達成すべき信頼:** 「表面的な会社じゃない」

**リスク:** 哲学が「言葉だけ」に見える
**対策:** プロダクトが哲学を証明している（言葉→証拠の順序）

---

### 5分: CONSIDERATION（検討）

**何が起きるか:** /about または /projects/[slug] を深読みする

**訪問者の心理状態（ペルソナ別）:**
- ユーザー: 「このアプリ、使ってみたい」
- 候補者: 「ここで働きたいかも」
- パートナー: 「一緒に何かできるかも」
- 共感者: 「フォローしよう」

**達成すべきアクション:** CTA を叩く

## Narrative Flow Design

### 比較分析: 提示された流れ vs 推奨案

**提示流れ（Problem → Mission → Approach → Projects → Vision → CTA）の評価:**

| 要素 | 評価 | 理由 |
|------|------|------|
| Problem から始める | ⚠️ リスクあり | 訪問者に「あなたは問題を抱えている」と言うことになる。押し付けになりうる |
| Mission を早めに | ✅ 良い | ENISHIは思想が差別化要因のため |
| Projects → Vision の順 | ⚠️ 逆推奨 | Visionを先に見せ、Projectsをその証拠にする方が論理的 |

**推奨フロー: HOOK → INSIGHT → STANCE → PROOF → VISION → CTA**

---

### 推奨 Narrative Flow（Top Page）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
HOOK — 縁 という言葉
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Hero H1: 「縁を、深く。」
Subhead: 「AI時代に、人間らしさのために。」
Visual: 暖かみのあるビジュアル（人と人のつながりを示唆）
CTA (secondary): プロジェクトを見る

目的: 会社名が「縁」であることを受け手に気づかせ、
      このサイトが「テクノロジー × 人間的つながり」の
      テーマであることを5秒で伝える

          ↓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PROOF — Projects が思想を証明する
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Featured Projects (2〜3枚)
Section title: 「作ってきたもの」または省略
Card: thumbnail / タイトル / tagline / statusバッジ
Link: すべてのプロジェクトを見る

目的: 「哲学は言葉だけ」ではなく、
      実際にプロダクトが存在することを証明する。
      言葉より先にプロダクトを見せることで信頼を先取り

          ↓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STANCE — ENISHIの立場
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Section: Philosophy / Vision
「私たちはAIを作る会社ではありません」
「テクノロジーは、人と人をつなぐためにある」
「だからすべてのプロダクトは、縁を軸に生まれます」

目的: 差別化。ENISHIがAI時代に何を信じているかを明示。
      この段階で「共感する/しない」が分かれる。
      分かれることは良いこと（ペルソナの絞り込み）

          ↓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PERSONA — 誰が作っているか
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
About Snippet
創業者/チームの1〜2行
写真またはアバター
Link: ENISHI-LABについて →

目的: 「会社」ではなく「人間」を見せる。
      スタートアップにとって創業者の顔が最大の信頼シグナル

          ↓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
VISION — 目指す世界への招待
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Vision section
「テクノロジーが人の縁を深める世界を、一緒に作りたい」
（または創業者の言葉で）

目的: 訪問者を「傍観者」から「共感者」へ変える。
      CTAへの心理的準備

          ↓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CTA — 縁をつなごう
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CTA Block
Primary: 「お問い合わせ」
Secondary: 「プロジェクトを見る」
（将来: ペルソナ別に分岐: 使う / 働く / 協業する）

目的: 共感を行動に変換する
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### なぜこの順序か

| 設計判断 | 理由 |
|---------|------|
| Problem から始めない | 訪問者に「問題がある」と仮定するのは失礼。好奇心から入る方が自然 |
| PROOF を STANCE より先に | 哲学の前にプロダクトを見せることで「言葉だけ」の印象を防ぐ |
| PERSONA を中盤に | 信頼の段階（30秒経過後）に顔を見せることで効果最大化 |
| VISION を最後から2番目に | 感情が高まった状態でVisionに触れると共感しやすい |
| CTA を最後に | 共感 → 行動の自然な流れ |

## Copy Tone Guidelines

### ENISHIの声（Brand Voice）

| 属性 | 定義 | 避けること |
|------|------|-----------|
| 誠実 | 現状を正直に伝える（beta表示、開発中の正直な説明） | 誇張・過剰な約束 |
| 温かみ | 人間の言葉で話す | 企業的・冷たいコーポレートスピーク |
| 知的 | 思想の深さを語れる | 空虚なバズワード（"革新的な""次世代の") |
| 簡潔 | 短い文で核心をつく | 冗長な説明 |
| 日本語的 | 余白・여백（yohaku）の美学 | 情報過多・詰め込みすぎ |

### コピーの禁止事項

```
❌ 「革新的な」「次世代の」「世界を変える」
❌ AI buzzwords（「生成AI」「LLM」をメインコピーで使わない）
❌ 「〜ソリューション」「〜プラットフォーム」
❌ 感嘆符の乱用（!）
❌ 根拠のない数字（"ユーザー数100万人目標！"）
```

### コピーの推奨

```
✅ 具体的な動詞（「作る」「つなぐ」「育てる」「守る」）
✅ 「縁」関連の語彙（縁・絆・つながり・出会い・想い）
✅ 一人称「私たち」（チームとしての声）
✅ 短い文と余白
✅ 現在形（今何をしているか）
```
