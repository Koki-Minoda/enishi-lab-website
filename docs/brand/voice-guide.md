# Brand Voice Guide

Last updated: 2026-06-28
Purpose: Messaging Architectureを文章・UI・Designに落とし込む実装ガイド

このドキュメントはENISHI-LABの「声」を定義する。
新しい思想の追加ではなく、Messaging Architectureの実装仕様書。

---

## Tone（声のトーン）

ENISHIの声は5つの属性で成り立つ。それぞれに「やること」と「やらないこと」がある。

### 1. 誠実（Sincere）

ENISHIは現状を正直に伝える。プロダクトが開発中ならそう言う。できないことは言わない。

```
✅ 「現在開発中です。リリースをお待ちください。」
✅ 「ベータ版として公開しています。」
❌ 「近日公開予定！」（根拠のない期待を煽る）
❌ 「世界中のユーザーに使われています」（未確認・誇大）
```

### 2. 温かみ（Warm）

ENISHIは人間の言葉で話す。企業的・官僚的な言葉は使わない。

```
✅ 「お問い合わせはこちらから。縁をつなぎましょう。」
✅ 「ご家族と一緒に試してみてください。」
❌ 「本サービスをご利用いただくにあたり…」
❌ 「当社は〜ソリューションを提供します」
```

### 3. 静けさ（Quiet / Calm）

ENISHIは叫ばない。感嘆符を乱用しない。主張より余韻を大切にする。

```
✅ 「縁を、深く。」（句点で終わる静かな力）
✅ 「テクノロジーを、縁のために。」
❌ 「今すぐダウンロード！！」
❌ 「圧倒的な体験をお届け！」
```

### 4. 知的（Intellectual）

ENISHIは思想の深さを持つ。ただし難解にしない。わかりやすく深い言葉を選ぶ。

```
✅ 「AIは手段。縁が、目的です。」（シンプルだが思想がある）
✅ 「テクノロジーは、人と人を切り離すためではなく、つなぐためにある。」
❌ 「革新的なAIアルゴリズムで最適化されたUXを実現」（空虚な専門用語）
❌ 「パラダイムシフトを起こす」（バズワード）
```

### 5. 人間中心（Human-centered）

ENISHIはプロダクトより人間を主語にする。技術の話をするときも、人が中心にいる。

```
✅ 「家族の笑顔が、このプロダクトを作った理由です。」
✅ 「大切な出会いを、忘れたくない。その気持ちから生まれました。」
❌ 「高度なアルゴリズムにより、パーソナライズされた体験を提供します」
❌ 「AIが最適な提案を自動生成します」
```

## Writing Principles（文章の原則）

### P1: 短く、深く

長い文は怠惰。短い文は読者への敬意。

```
✅ 「縁を、深く。」（2語）
✅ 「テクノロジーを、縁のために。」（6語）
❌ 「ENISHIラボはAI技術を活用したプロダクト開発を通じて、人と人とのつながりを
    深めることを目指したスタートアップ企業です。」（長すぎ）
```

### P2: 具体 > 抽象

抽象的な約束より、具体的な体験を語る。

```
✅ 「家族全員で今日のクエストをクリアしよう。」
✅ 「3年前に出会ったあの人の名前、覚えていますか。」
❌ 「人間関係の質を向上させるソリューション」
❌ 「コミュニケーションを最適化」
```

### P3: ポジティブで定義する

「〜ではありません」は使わない。何であるかを語る。

```
✅ 「テクノロジーを、縁のために使う会社です。」
❌ 「AIを作る会社ではありません。」
✅ 「思想から生まれたプロダクトを作ります。」
❌ 「単なるツール会社ではありません。」
```

### P4: 読者を主語にしない

「あなたには問題があります」という出発点は失礼。読者が自分で気づく余白を作る。

```
✅ 「家族の絆を、もっと深めたいと思ったことはありませんか。」（質問形式）
✅ 「大切な出会いを、記録する場所。」（役割を語る）
❌ 「あなたは大切な人間関係を管理できていますか？」（責める）
❌ 「家族との絆が希薄になっている現代社会において…」（問題を押し付ける）
```

### P5: 余白を恐れない

書かないことも、書くことと同じくらい重要。

```
✅ 短い段落 + 空白行
✅ 1セクション = 1メッセージ
❌ 情報を詰め込んだ長い段落
❌ 全ての余白を埋めようとする
```

### P6: 日本語を主とする

英語は必要な場面のみ。プロダクト名・技術用語・固有名詞を除き、日本語で書く。

```
✅ プロダクト名: Family Quest（英語名はそのまま）
✅ 「App Storeでダウンロード」（固有名詞）
❌ 「Our Mission is to connect people.」（Heroを英語にする）
❌ 「User Experienceを向上させる」（カタカナ英語の乱用）
```

## Recommended Expressions（推奨表現）

### 縁に関する語彙

```
縁 / ご縁        → 使う。ENISHIの核心語
絆              → 家族・友人との深い繋がり
つながり         → 縁より現代的。日常の繋がり
出会い          → 新しい縁の始まり
きっかけ         → 縁の入り口

縁をつなぐ       → 動詞としての縁
縁を育てる       → 時間をかけて深まる縁
縁を記録する     → ご縁ノートの文脈
縁を守る        → Family Questの文脈
```

### 推奨動詞（温かみがある）

```
作る / つくる    → プロダクト開発
つなぐ          → 人と人
育てる          → 関係・家族
守る           → 日常の価値
深める          → 縁・理解
記録する        → 記憶・ご縁
```

### 推奨フレーム

```
「〇〇の縁」               → カテゴリ名
「〜のために作りました」    → プロダクトの理由
「一緒に〜」               → CTAの人間的な誘い
「私たちは〜」             → We statements（企業ではなく人）
```

## Avoid Expressions（避ける表現）

### AIバズワード（ENISHIの立場と矛盾・陳腐化）

```
❌ AI革命 / AI活用（AIを売り文句にしない）
❌ 生成AI / LLM（技術用語をメインコピーにしない）
❌ ChatGPT〜みたいな
❌ AI時代のソリューション
❌ DX推進
```

### 誇大表現（誠実さと矛盾）

```
❌ 世界を変える / 社会を変革する
❌ 最先端の / 革新的な / 圧倒的な
❌ No.1 / ナンバーワン
❌ 唯一の / 史上初の（根拠なし）
❌ Disrupt / ゲームチェンジャー
```

### カタカナ乱用（温かみと矛盾）

```
❌ ソリューション → 解決策、または具体的に言う
❌ プラットフォーム → （具体的な機能で説明）
❌ エコシステム → （使うなら定義してから）
❌ コミットメント → 約束、誓い
❌ シナジー → 相乗効果（または使わない）
```

### 符号の乱用（静けさと矛盾）

```
❌ ！！（感嘆符の重複）
❌ ？？（疑問符の重複）
❌ 〜〜（語尾の伸ばし）
❌ ✨🎉🚀（絵文字をコピーに使う）
```

### Hero Section

```
H1:   縁を、深く。

H2:   人と人をつなぐプロダクトを、AI時代に届ける。

Body: テクノロジーを、縁のために使う会社です。

CTA1: プロジェクトを見る
CTA2: お問い合わせ
```

**別案A（よりストーリー性を出す場合）:**
```
H1:   縁を、深く。
H2:   家族の絆から、大切な出会いまで。
Body: ENISHIは、テクノロジーで縁を深めるプロダクトを作っています。
```

---

### About Section（Origin Story）

```
「縁」という言葉を社名に選びました。

人と人の出会い、家族との日常、忘れたくない記憶。
テクノロジーはそれを消すためではなく、深めるためにあると信じているからです。

AI時代を、人間らしく生きるために。
私たちは今日も、縁を育てるプロダクトを作っています。
```

---

### About Section（Mission）

```
テクノロジーを、縁のために使う。

AIを含むあらゆる技術は、私たちにとって手段です。
目的はひとつ。人と人の縁を、深めること。

その信念から生まれたのが、ENISHI-LABのすべてのプロダクトです。
```

### Project Card（Family Quest）

```
Title:   Family Quest
Tagline: 家族の冒険で、絆を深める。

Card描写（ProjectCard上に表示しないが、詳細ページで使用）:
「今日のクエスト、一緒にクリアしよう。」
家族の「やること」をゲームに変えると、会話が変わります。

Status: active
Category: 家族の縁
```

```
Status: development
Category: 出会いの縁
```

### Form Labels（日本語で、人間的に）

```
お名前       （not: Name / 氏名）
メールアドレス（not: Email）
お問い合わせ種別:
  ビジネスについて
  採用について
  プレスについて
  その他
メッセージ   （not: お問い合わせ内容）

[送信する]   （not: Submit / 送信）
```

---

### Error Messages（誠実に、温かく）

```
入力内容を確認してください。
（not: Invalid input / エラーが発生しました）

メールアドレスの形式が正しくありません。
（not: Invalid email format）

送信に失敗しました。もう一度お試しください。
（not: Error 500 / Request failed）
```
